# Security Policy

## Supported Versions

Security fixes are currently intended for the latest state of the default branch.

## Reporting a Vulnerability

Please do not open a public issue for an unpatched security vulnerability.

Use the repository's private security reporting or maintainer contact channel if
it is available on the hosting platform. If a private advisory workflow has not
yet been configured, contact the maintainers directly through the repository
owner contact shown on the hosting platform and include:

- a short description of the issue
- affected files or components
- impact assessment
- reproduction steps
- any suggested mitigation

We will acknowledge a valid report as quickly as practical and coordinate on a
fix and disclosure timeline.
